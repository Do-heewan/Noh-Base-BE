# 기능 명세 (MVP)

> 출처: `의학용어_단어장_앱_MVP_기획서.docx`. 이 문서는 그 핵심을 개발 기준으로 요약한 것이다.

## 한 줄 정의

교수가 준 이번 주 시험범위를, 사진/PDF 한 장으로 단어장으로 만들어 외우게 해주는 앱.

## 해결하는 문제

간호학과 학생은 매주 ~200개 영어 의학용어를 외운다. 진짜 마찰점은 암기 자체가 아니라
**시험범위를 단어장으로 '옮겨 적는' 시간**이다. 이 시간을 0에 가깝게 만드는 것이 핵심 가치다.

## 타겟 / 진입점

- **1차 타겟**: 간호학과 3학년 전체 (반복적·명확한 시험 트리거).
- **진입점(beachhead)**: 25세 이상 만학도 그룹 — 지불의향(WTP)이 높다는 *가설*. 제품은 전 학년이 쓰도록 설계.
- **검증 장치**: 가입 시 입학 유형(현역/편입/만학) 수집 → 유형별 유료 전환율·잔존율 비교.

## 핵심 루프 (반드시 매끄럽게)

1. **입력** — PDF 업로드 또는 카메라 스캔으로 시험범위를 가져온다.
2. **추출** — 영어 의학용어를 자동 추출.
3. **자동 완성** — 자체 의학용어 DB 대조로 뜻·어원을 채움 (미수록 단어는 LLM 폴백).
4. **검수** — 사용자가 결과를 빠르게 확인·수정하고 단어장으로 저장. **(품질 안전장치, 절대 생략 불가)**
5. **학습** — 객관식·주관식 랜덤 퀴즈로 암기, 오답 자동 누적.
6. **지속** — 학습 캘린더·스트릭으로 꾸준함 유도.

## 입력 경로 우선순위

| 경로 | 처리 | 우선순위 |
| --- | --- | --- |
| 디지털 PDF | 텍스트 레이어 직접 추출 (PyMuPDF) | 1순위 — 먼저 완성해 핵심 루프 검증 |
| 카메라 스캔 | OCR | 2순위 |
| 이미지 PDF | 텍스트 레이어 없음 → OCR 폴백 | 카메라용 OCR 엔진으로 동시 커버 |

PDF 처리는 처음부터 **'텍스트 레이어 있으면 직접 추출, 없으면 OCR 폴백'** 분기를 갖춘다.

## 자동 완성 — 자체 DB 방식

- 의학용어는 어휘가 사실상 닫혀 있어, 한 번 구축하면 재사용되는 자산. 매칭 1건당 비용 ≈ 0.
- DB 미수록 단어는 LLM 폴백, **사용자 검수 결과를 DB에 다시 적립**해 점진 보강.
- 검수 단계는 ①품질 보증 ②DB 개선용 데이터 수집 ③'내가 다듬은 단어장' 신뢰감을 동시에 달성.

## 화면 흐름 (한 화면 = 한 결정)

| # | 화면 | 행동 | 비고 |
| --- | --- | --- | --- |
| 1 | 홈/단어장 목록 | '새 단어장 만들기' | 스트릭·오늘 학습 현황 표시 |
| 2 | 입력 방식 선택 | PDF 업로드 / 카메라 | 무료 사용자 잔여 변환 횟수 표시 |
| 3 | 추출·처리 중 | 대기 | 추출 + DB 매칭 자동 수행 |
| 4 | 검수 화면 | 단어·뜻 확인/수정, 빈칸 보완 | **MVP 품질의 핵심** |
| 5 | 저장 완료 | 단어장 이름 지정·저장 | 바로 '학습 시작' 유도 |
| 6 | 퀴즈 | 객관식·주관식 랜덤 | 오답 자동 누적 |
| 7 | 학습 결과 | 정답률, 스트릭 갱신 | 오답만 다시 풀기 |

## 성공 지표

① 스캔/업로드 시도율, ② **검수 완료해 저장한 비율(핵심 전환)**, ③ 저장 후 퀴즈 1회+ 비율,
④ **다음 주 새 단어장 재생성 재방문율**. ②·④ 가 핵심 가설의 직접 검증 지표.

## 수익 모델

- 비용 동인은 저장 용량이 아니라 **스캔·변환 횟수**(OCR/LLM 호출). 한도도 변환 횟수에 건다.
- **Free**: 단어 저장 넉넉(300–500+), 변환 월 3–5회, 퀴즈·캘린더·스트릭 전체 제공.
- **Pro(예: 월 9,900원)**: 변환 무제한, v2 기능(팀 공유·고급 통계). 구체 수치는 실비용 데이터 확보 후 확정.

## MVP 제외 (구현하지 않음)

팀 구성·단어장 공유(v2 확산 장치), 손글씨 OCR, 게임 모드, 발음 듣기 등 부가 학습.
